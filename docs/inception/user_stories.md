# ユーザーストーリー: Tesla × Alexa 連携

本ドキュメントは [overal_plan.md](./overal_plan.md) の Phase 1「ユーザーストーリー洗い出し」の成果物です。

---

## エピック一覧

E の番号順が優先順位（E1 → E2 → E3 → E4 → E5）。

| ID | エピック | 概要 |
|----|----------|------|
| E1 | Alexa とテスラを安全に連携する | Account Linking による認証・トークン連携の基盤 |
| E2 | Alexa からテスラの車内温度を確認する | 音声で車内・外気温を聞く（初期スコープ） |
| E3 | Alexa からテスラのエアコンを操作する | 音声でエアコンを ON にする（初期スコープ） |
| E4 | Alexa スキルとして公開する | ストア公開・他ユーザー利用 |
| E5 | Alexa からテスラのその他操作を行う | 優先度 高・中・低の機能（詳細は作る時に決める） |

## エピック・ユーザーストーリー詳細

### 【エピック E1】Alexa とテスラを安全に連携する

- **誰のために**: テスラオーナー（Alexa 利用者）
- **何を実現するか**: Alexa からテスラ API を呼ぶために、本人確認済みの Tesla アカウントと安全に紐付ける。
- **備考**: Account Linking（OAuth2）採用済み。Lambda は Alexa から渡されるアクセストークンで Tesla Fleet API を呼ぶ。連携済みユーザーがスキルを呼んだときのトークン渡し・API 成功は E2-1 / E3-1 の受け入れで検証する。

#### ストーリー E1-1: 初回で Alexa とテスラを連携する

- **役割**: テスラオーナーとして
- **やりたいこと**: 初めてスキルを使うときに、Alexa アプリ（または画面付きデバイス）で「テスラと連携」を完了したい
- **得られる価値**: 自分のテスラアカウントだけが操作対象になり、他人の車を触らないようにできる
- **受け入れ条件**:
  - Alexa 側で Account Linking を開始すると、Tesla の認可画面（ログイン／同意）に遷移する
  - 認可完了後、Alexa に戻り「連携が完了しました」と分かる
  - 連携後、スキル発話で Lambda が Alexa からアクセストークンを受け取れる

---

### 【エピック E2】Alexa からテスラの車内温度を確認する

- **誰のために**: テスラオーナー
- **何を実現するか**: 家の中や手が離せないときに、音声だけで「今の車内・外気温」を把握できるようにする。
- **備考**: 初期スコープ。Smart Home Skill の `TemperatureSensor` インターフェースで実現。車両が sleep の場合は wake_up → ポーリングの考慮が必要（既存 `get_vehicle_data.py` の知見を流用）。

#### ストーリー E2-1: 車内温度を音声で聞く

- **役割**: テスラオーナーとして
- **やりたいこと**: 「アレクサ、テスラの温度は？」と聞くと、現在の車内温度を音声で知りたい
- **得られる価値**: 出かける前にスマホを触らず、声だけで車内が暑い/寒いか判断し、エアコンを事前に ON にするか決められる
- **受け入れ条件**:
  - Smart Home Skill の `TemperatureSensor` インターフェースで状態レポートを返す
  - Tesla Fleet API の `vehicle_data?endpoints=climate_state` を呼び、`inside_temp` を取得する
  - 車両が sleep の場合は wake_up してから取得する（既存ロジックを流用可能）
  - Alexa が「テスラの温度は〇〇度です」と返す（日本語 ja-JP）
  - エラー時は適切なエラーレスポンスを返す

#### ストーリー E2-2: 外気温を音声で聞く（オプション・優先度低め）

- **役割**: テスラオーナーとして
- **やりたいこと**: 「テスラの外の温度は？」と聞くと、外気温を音声で知りたい
- **得られる価値**: 車に乗る前に外の気温を把握できる
- **備考**: E2-1 と同じ API（`climate_state` → `outside_temp`）で取得可能。初期リリースでは E2-1 のみでも可。

---

### 【エピック E3】Alexa からテスラのエアコンを操作する

- **誰のために**: テスラオーナー
- **何を実現するか**: 出発前に、音声だけでエアコンを ON にし、車内を快適な温度にしておく。
- **備考**: 初期スコープ。API は `POST .../command/auto_conditioning_start`。

#### ストーリー E3-1: エアコンを ON にする

- **役割**: テスラオーナーとして
- **やりたいこと**: 「アレクサ、テスラのエアコンをつけて」と言うと、リモートでエアコンが ON になる
- **得られる価値**: 夏は涼しく、冬は暖かく、乗る前に車内を準備できる
- **受け入れ条件**:
  - 発話例に対して、Tesla Fleet API の `command/auto_conditioning_start` を実行する
  - 成功時は Alexa が「エアコンをつけました」と返す（ja-JP）
  - 失敗時（車両不明・API エラーなど）は「エアコンを付けられませんでした」など分かるメッセージを返す

#### ストーリー E3-2: エアコンを OFF にする（後で詳細化）

- **役割**: テスラオーナーとして
- **やりたいこと**: 「エアコンを消して」でリモートでエアコンを OFF にしたい
- **得られる価値**: つけっぱなしを防ぎたいときに、音声で消せる
- **備考**: 優先度は E3-1 の次。受け入れ条件は後で詳細化可。

---

### 【エピック E4】Alexa スキルとして公開する

- **誰のために**: 他のテスラオーナー（および自分が複数デバイス・アカウントで使う場合）
- **何を実現するか**: 個人利用で安定したら、Alexa Skills としてストアに公開し、他ユーザーも「テスラ連携」して使えるようにする。
- **備考**: 審査要件・プライバシーポリシー・説明文などは別タスクで整理。

#### ストーリー E4-1: スキル公開の準備と申請（後で詳細化）

- **役割**: 開発者として
- **やりたいこと**: スキルを公開するために、審査に通る情報（説明、サンプル発話、プライバシー等）を揃えたい
- **得られる価値**: 他のテスラオーナーがスキルを有効化して使えるようになる
- **備考**: 受け入れ条件は公開方針が固まってから記載。

---

### 【エピック E5】Alexa からテスラのその他操作を行う

- **誰のために**: テスラオーナー
- **何を実現するか**: 優先度 高・中・低の機能を、音声から実行・確認できるようにする。詳細は各ストーリーを作る時に決める。

#### ストーリー（タイトルのみ・優先度順）

**優先度：高**

- エアコンを OFF にする
- バッテリー残量（%）を聞く（※ Smart Home で自然な発話が難しい場合、Custom Skill 併用を検討）
- 充電中か / あと何分でフルか（状態確認、同上）
- 走行距離（オドメーター）を聞く（同上）
- 充電を開始する
- 充電を停止する
- トランク（リア/フロント）を開閉する

**優先度：中**

- 温度を設定する
- シートヒーター / ステアリングヒーター
- 車内・外気温を聞く（状態確認）
- ロックされているか確認（状態確認）
- ドアを施錠する
- セントリーモード ON/OFF
- ウィンドウを換気/閉める

**優先度：低**

- ホーンを鳴らす
- ライトをフラッシュする

## スキル種類

- **採用**: **Smart Home Skill**
  - 呼び出し名（invocation name）が不要。「アレクサ、テスラのエアコンをつけて」とだけ言えば動作する。
  - 操作系（ON/OFF、温度設定、施錠、トグル系）の大半を自然な発話でカバーできる。
  - Smart Home のインターフェース（`PowerController`, `ThermostatController`, `TemperatureSensor`, `LockController` 等）を利用。
- **将来検討**: 情報確認系（バッテリー残量、充電状態、オドメーター等）は Smart Home の定型に合わないため、Custom Skill の併用を検討する。
